[ PACKAGIST HELPER ]

Criando/Publicando um projeto no Packagist

- Instalar o composer e o git
- Ter uma conta no Packagist
- Ter uma conta no GitHub
- Criar o projeto no GitHub
	- Exemplo: https://github.com/vendor/project.git
- Clonar o projeto
	- git clone https://github.com/vendor/project.git .
- Criar o arquivo package.json
	- composer init
		- Exemplo:
			{
			    "name": "vendor/project",
			    "type": "project",
			    "description": "Project descriptin...",
			    "keywords": ["key1","key2"],
			    "homepage": "http://www.webpage.com/page",
			    "license": "MIT",
			    "version": "1.0.0",
			    "authors": [
			        {
			            "name": "FirstName LastName",
			            "email": "vendor@email.com"
			        }
			    ],
			    "require": {
			    },
			    "require-dev": {
			    },
			    "autoload": {
			        "psr-4": {
			            "Namespace\\": "src/"
			        }
			    },
			    "scripts": {
			    },
			    "suggest": {
			    }
			}
- Estrutura de pastas do projeto (Exemplo):
    project/
        .gitignore
        composer.json
        src/
       		ProjectTest.php
	
	- O arquivo .gitignore deve ter a pasta /vendor/
- Enviar as alterações para o repositorio git
	git add .
	git commit -m "Initial Commit"
	git push -u origin master
- Publicar o repositorio no Packagist
	Efetuar login: https://packagist.org/login/
	Enviar pacote: Submit package => Check => Submit
	Configurar o Token Packagist no GitHub:
		GitHub > Project > Settings > Webhooks > Add Webhook
			Preencher formulario:
				Payload URL: https://packagist.org/api/github?username=seu_usuario_packagist
				Content type: application/json
				SSL verification: Enable SSL verification
				Secret: (Token Packagist)
					O token pode ser obtido em https://packagist.org/profile/, clicando em Show API Token
- Instalar/Utilizar o pacote
	Via package.json:
    "require": {
        "slim/slim": "4.2.0",
        "rain/raintpl": "3.1.1",
        "ext-json": "*",
        "huntercodexs/phphunterkernel": "1.0.0"
    }
	composer update
    Via comando:
	composer require vendor/project
	composer require vendor/project:dev-master (Para projetos em desenvolvimento e sem release/versão)
- Criar uma release do projeto no GitHub
	acessar repositorio do projeto
	clicar em Release no lado direito da tela
	configurar a release, exemplo:v1.0.0
	publicar
- Projeto/Pacote sem Release
	Configurar no projeto que vai usar o pacote o minimum-stability como dev
		"minimum-stability": "dev",
		"prefer-stable": true,
