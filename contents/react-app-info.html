Criando um projeto REACT JS

Inicialização

Para iniciar um projeto em React JS, crie uma pasta dedicada ao projeto 

mkdir /home/user/devel/projects/react/project_name
cd /home/user/devel/projects/react/project_name

Logo apos criar e acessar a pasta criada, executar o comando de criação do projeto modelo 

npx create-react-app .

Verificar, e editar se for o caso, as informações no arquivo package.json

Iniciar serviço/servidor node/react 

npm start

Acessar o projeto criado pelo browser 

http://localhost:3000
http://192.168.15.13:3000

Opcionalmente, pode se criar o projeto no GitHub e depois clona-lo, assim como também dar as devidas permissões de escrita no projeto

Agora é só abrir o projeto em uma IDE ou Editor de preferência e programar a aplicação

Estrutura de pastas

public/ # Pasta publica onde ficam os arquivos publicos do projeto, como o index.html
src/ #Source do Projeto, onde ficam os componentes iniciais como o App
node_modules/ # Modulos NodeJS

Configurações Extras

Instalar icones do react

npm install react-icons --save

Trabalhar com rotas

npm install react-router-dom --save

Trabalhar com requisições

npm install axios --save

Descrição das pastas

build/

A pasta build é onde fica os arquivos "compilados" ou melhor "buildados" com o comando npm run build, sendo que esses arquivos podem ser acessados ou incorporados em qualquer projeto web tradicional, não necessitando de um servidor web do tipo node.

node_modules/

Essa é a pasta onde o npm faz as instalações dos módulos que são baixados de acordo com as necessidades do projeto utilizando o comando npm install.

package.json

Nesse arquivo estão localizados todas as configurações do projeto, assim como as dependências do mesmo, exemplo: 

    "dependencies": {
        "@testing-library/jest-dom": "^5.14.1",
        "@testing-library/react": "^11.2.7",
        "@testing-library/user-event": "^12.8.3",
        "axios": "^0.21.1",
        "react": "^17.0.2",
        "react-dom": "^17.0.2",
        "react-router-dom": "^5.2.0",
        "react-scripts": "4.0.3",
        "web-vitals": "^1.1.2"
    },

package-lock.json

O package-lock.json não representa uma informação muito relevante, apenas mostra como se fosse um log os pacotes ou modulos que foram instalados no projeto (mas não estão todos sendo usados necessariamente)

public/

Como o nome da pasta sugere, nela ficam os arquivos públicos que serão acessados pelo browser para renderizar o conteúdo da aplicação, o principal arquivo dessa pasta é o index.html.

Existe uma div nesse arquivo identificada pelo id root, e por padrão é nessa div que todo o conteudo da aplicação é renderizado pelo React, importante não alterar esse id, ou manter sincronizado com o App.

src/

Como ultima e não menos importante temos a pasta src (source) onde todo o conteúdo programável da aplicação é feito. Nessa pasta tempo uma estrutura padrão inicial, como abaixo:

O arquivo inicial é o index.js responsável por carregar na aplicação todo o conteúdo programado através da importação de bibliotecas React e do script principal App.jsx, conforme exemplo abaixo: 

import React from 'react';
import ReactDOM from 'react-dom';

import './index.css';
import App from './App';

ReactDOM.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
  document.getElementById('root')
);

Os arquivos CSS dispensam explicações, e o script App.jsx é o script central da aplicação, onde todo recurso do app é concentrado, utilizado e gerido.

Na pasta componentes estão localizados os componentes do app, como botões, listas, menus, telas entre outros.

Fluxo de Execução de um projeto REACT JS

Inicialização

O processo de execução do app pode ser exemplificado a seguir: 

index.html < index.js < App.jsx, App.css < components/{Component1, Component2...}

Ao iniciar uma aplicação React todos os recursos previamente programados são incorporados/renderizados no arquivo index.html dentro do elemento html com id "root"...

O script App.jsx contido na pasta src é quem ajunta os demais scripts que por sua vez representam os componentes da aplicação ou app.

Boas praticas e definições

É uma boa pratica e até mesmo muito correto buscar sempre que possível o desenvolvimento de forma a ter componentes reutilizaveis no projeto, por exemplo: um botão ? SIM, criando scripts especificos para ele assim como seu arquivo de estilos CSS, conforme exemplo abaixo:

App.jsx 
import React from 'react';

import "./Button.css"

const Button = ({ children, onClick }) => {
    return <button onClick={onClick} className="add-task-button">
        {children}
    </button>;
};

export default Button;
App.css
